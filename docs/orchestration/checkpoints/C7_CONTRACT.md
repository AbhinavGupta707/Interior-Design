# C7 Contract — Native iOS RoomPlan capture and secure sync

## Authority, outcome and scope

- Checkpoint: C7
- Immutable predecessor: C6 ledger-close commit `96d0a72`
- Outcome: a supported physical LiDAR iPhone/iPad can perform resumable single-room and multi-room/structure capture, retain immutable RoomPlan evidence, securely upload a versioned package, and receive a bounded canonical-shaped proposal or explicit abstention. Unsupported devices continue through the existing plan/photo/manual route.
- Contract versions: `c7-capture-session-v1`, `c7-capture-package-v1`, `c7-roomplan-normalized-v1`, `c7-capture-proposal-v1`
- Scope boundary: C7 captures and converts RoomPlan evidence. It does not reconstruct ordinary RGB media (C8), fuse plan/scan/reconstruction evidence or resolve discrepancies (C9), compile a browser scene (C10), infer hidden construction, establish a professional survey, or mutate canonical state directly.

The authoritative product and safety rules remain `AGENTS.md`, the accepted C4-C6 contracts and ADRs, and the active blue-sky execution plan. This contract narrows C7 implementation; it does not relax provenance, rights, tenant, typed-mutation or professional-status invariants.

## Primary-source decisions

- Apple documents RoomPlan as an iOS/iPadOS camera-and-LiDAR parametric capture framework. `RoomCaptureSession.isSupported` remains the runtime capability boundary; Simulator and Mac Catalyst processing cannot prove capture. [RoomPlan framework](https://developer.apple.com/documentation/roomplan), [ARKit support and permission](https://developer.apple.com/documentation/arkit/verifying-device-support-and-user-permission)
- Multi-room capture uses one compatible world space: stop a room with `stop(pauseARSession: false)`, retain each processed `CapturedRoom`, continue the AR session, and call async `StructureBuilder.capturedStructure(from:)`. The merge must abstain when rooms do not share compatible world space. [Scanning one structure](https://developer.apple.com/documentation/roomplan/scanning-the-rooms-of-a-single-structure), [StructureBuilder](https://developer.apple.com/documentation/roomplan/structurebuilder)
- `CapturedRoom`, `CapturedStructure` and `CapturedRoomData` are `Encodable`; raw data may be serialized for deferred processing. C7 preserves those encodings and USDZ as immutable artifacts, but never treats their opaque representation or appearance as the canonical store. [CapturedStructure](https://developer.apple.com/documentation/roomplan/capturedstructure), [CapturedRoomData encoding](https://developer.apple.com/documentation/roomplan/capturedroomdata/encode%28to%3A%29)
- Native guidance consumes RoomPlan's exact instruction stream (`normal`, move closer/away, turn on light, slow down, low texture). A separate versioned product heuristic may summarize capture quality, but cannot claim completeness or measurement accuracy from instruction counts alone. [RoomCaptureSession instructions](https://developer.apple.com/documentation/roomplan/roomcapturesession/instruction)
- Relocalisation is recoverable, not guaranteed. During interruption the app attempts a bounded return-to-origin flow, exposes restart, and never silently joins incompatible coordinate spaces. A saved `ARWorldMap` is local resume state only and is excluded from the server package in C7. [Relocalisation lifecycle](https://developer.apple.com/documentation/arkit/arsessionobserver/sessionshouldattemptrelocalization%28_%3A%29), [ARWorldMap](https://developer.apple.com/documentation/arkit/arworldmap)
- RoomPlan surfaces now include polygon corners, completed edges and optional curves. The normalized contract preserves those observations rather than flattening them silently; unsupported conversion becomes a finding/unknown/abstention. [CapturedRoom surface](https://developer.apple.com/documentation/roomplan/capturedroom/surface)

## Frozen product and data decisions

1. The app remains native Swift 6/SwiftUI with minimum iOS 17.0. It uses RoomPlan/ARKit on supported physical hardware and an injected protocol/fixture engine for Simulator state tests.
2. A server-issued capture brief pins project, session, mode, expiry, instruction version and rights. Service processing must be explicitly allowed; training use is fixed to `denied` for C7.
3. Single-room and structure modes are explicit. Structure mode preserves one shared/relocalised AR world origin, one ordered room list and the `CapturedStructure` merge result. Rooms from a restarted/incompatible world space cannot be labelled as one structure.
4. Local capture state is a strict state machine with permission, ready, scanning, interrupted, relocalising, room review, structure review, packaging, offline/paused upload, uploading, processing, proposed, abstained, cancelled and safe failure views. Background termination can resume upload from persisted checksums but cannot resume an authoritative live sensor session without relocalisation/review.
5. Local resume files use iOS data protection; authentication belongs in Keychain. Capture artifacts, object keys, signed URLs, tokens, world maps and raw JSON never enter analytics/crash messages or Git. No captured camera frames are retained by C7.
6. The package contains exact hashes and bounded descriptors for per-room encoded `CapturedRoom`, optional encoded `CapturedRoomData`, one normalized integer RoomPlan artifact, one quality manifest, a structure encoding for structure mode, and optional USDZ. Files upload independently through short-lived, checksum-bound multipart sessions so background sync can resume without building a huge archive or loading source data into memory.
7. The app converts RoomPlan floating values at the edge into integer micrometres, nanounit rotation bases and nanoradians using one documented half-away-from-zero rule. The immutable Apple encoding remains available to audit the conversion. The server rejects non-finite/out-of-range transforms, duplicate IDs, missing parents, unknown room references, incompatible package/session/project identity and hash/version mismatches.
8. `CapturedRoom` confidence is classification confidence, not metric accuracy. Reference measurements are explicit user assertions between two source entities; they never retroactively become observed sensor truth.
9. Source artifacts are immutable in the source bucket. Normalized/converter/proposal artifacts identify exact source hashes, app/OS/RoomPlan versions, heuristic/converter versions and manifest hashes. Revoke/cancel/rights-change paths fail closed before worker lease and publication.
10. Conversion produces either an immutable `c7-capture-proposal-v1` existing-state snapshot with source mappings/findings/unknowns or an explicit abstention. It does not call C5 preview/commit. C9 owns plan/scan/reconstruction registration, discrepancy review and any user-confirmed canonical mutation.

## Bounded contract and route inventory

The shared schemas in `packages/contracts/src/c7.ts` freeze:

- at most 64 rooms, 256 artifacts, 10,000 surfaces, 10,000 objects, 100 reference measurements and 2 GiB total source bytes;
- a 512 MiB per-artifact ceiling, 8 MiB multipart plan and 10,000-part cap;
- exact artifact kinds/media types, room scoping, source SHA-256 and byte sizes;
- device/app/OS information without a user/device advertising identifier;
- a complete quality manifest and strictly increasing capture interval no longer than six hours;
- integer coordinate, transform, curve, polygon, confidence, parent and story fields;
- strict proposal/abstention output and a minimum proposal confidence of 60; and
- no secret, signed URL, object key, local path, raw source payload or world map in public/audit contracts.

Frozen public routes:

- `GET|POST /v1/projects/:projectId/capture-sessions`
- `GET /v1/projects/:projectId/capture-sessions/:captureSessionId`
- `POST /v1/projects/:projectId/capture-sessions/:captureSessionId/cancel`
- `POST /v1/projects/:projectId/capture-sessions/:captureSessionId/retry`

## Required checkpoint gate

1. `pnpm verify`, focused C7 contract/authz tests, Swift format/strict-concurrency build and all pre-existing suites pass. Cross-language golden JSON is accepted byte-for-byte by Swift and TypeScript; unknown fields, floating normalized coordinates and duplicate/missing identifiers reject.
2. A clean disposable PostGIS database applies C1-C7. Live API/storage/worker tests prove exact tenant/project/session/package scope, multipart replay/conflict, checksum/byte/kind binding, rights withdrawal, cancellation during upload/process, retry fencing, immutable terminal results, one proposal/abstention and no canonical mutation.
3. Adversarial tests cover IDOR, malformed/oversized JSON, hostile strings, object/path/URL confusion, artifact substitution, USDZ signature/type mismatch, duplicate/out-of-order parts, expired signed URLs, interrupted completion, incompatible world space, extreme transforms/counts, log redaction and worker resource limits.
4. XcodeGen regeneration is stable. Simulator unit and UI tests cover capability/permission messaging, project/brief loading, every local state, instruction/quality/reference-measure presentation, room/structure review, interruption/restart fixtures, offline/background upload reconciliation, expiry/forbidden/cancel/retry and Dynamic Type/VoiceOver. Simulator results are always labelled non-RoomPlan.
5. A generic physical-iOS no-signing build compiles RoomPlan/ARKit code. Computer Use launches the integrated Simulator app and verifies the actual user route, error recovery, orientation/narrow layouts and accessible fallback without claiming sensor behavior.
6. The physical-device gate records device model, OS, app/release SHA and RoomPlan result for permission denial, one room, at least two connected rooms merged as one structure, interruption/relocalisation or explicit safe restart, offline/resumable upload, low-light/low-texture guidance and source/proposal hash linkage. A named field protocol may record a case as failed or `NOT RUN`; Simulator/generic device compilation cannot turn it into a pass.
7. No GPU gate applies. No paid provider, API key, customer capture or cloud service is required for deterministic implementation and integration gates. Fixture evidence is visibly synthetic and can never be presented as a physical field capture.
8. The ledger records task IDs, exact model/reasoning, worker/merge/product SHAs, commands/counts, live database/object/API evidence, Simulator and physical-device status, Computer Use observations, integration repairs, limitations and final checkpoint state before C8 opens.

DQ-020 therefore remains unresolved and the field matrix is presently `NOT RUN`, not a pass.
